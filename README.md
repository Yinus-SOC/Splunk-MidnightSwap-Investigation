# Splunk MidnightSwap SOC Investigation

## Project Overview

This project demonstrates a Security Operations Center (SOC) threat investigation using Splunk Enterprise. The objective was to analyze the MidnightSwap security dataset, identify malicious activity, investigate attack patterns, and present the findings through an interactive SOC dashboard.

---

## Objectives

- Deploy Splunk Enterprise as a SIEM platform
- Analyze security events using SPL
- Investigate suspicious IP addresses
- Identify targeted usernames
- Detect attack categories
- Build an interactive SOC dashboard
- Document investigation findings

---

## Tools Used

- Splunk Enterprise
- Kali Linux
- Oracle VirtualBox
- Search Processing Language (SPL)
- MidnightSwap Dataset

---

## Skills Demonstrated

- SIEM Administration
- Log Analysis
- Threat Hunting
- SOC Monitoring
- SPL Query Development
- Dashboard Creation
- Incident Investigation

---

## Investigation Process

1. Installed and configured Splunk Enterprise on Kali Linux.
2. Imported the MidnightSwap dataset.
3. Parsed and indexed security events.
4. Developed SPL searches to identify malicious activity.
5. Investigated suspicious IP addresses and targeted usernames.
6. Built an interactive SOC dashboard to visualize security events.
7. Documented findings and security recommendations.

---

## Key Findings

- Brute-force attacks were the most common attack type.
- Multiple phishing-related activities were detected.
- Several IP addresses repeatedly generated suspicious events.
- Some usernames were targeted more frequently than others.
- Attack activity showed noticeable spikes over time.
